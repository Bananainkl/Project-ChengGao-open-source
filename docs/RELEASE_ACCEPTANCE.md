# 澄稿安装包可分发验收

本清单用于保证功能不是只在开发电脑上成立。所有应用代码、资源、依赖或发行脚本修改，都必须按影响范围完成相应项目；没有证据的项目不得写成已通过。

## 1. 构建与安装包

- 递增 `VERSION` 和 `BUILD_NUMBER`，更新 `CHANGELOG.md`。
- 运行全量测试并记录测试数量。
- 使用正式发行脚本生成当次 DMG，不复用旧 App 或旧 DMG。
- 挂载当次 DMG，确认根目录同时存在“澄稿.app”“Applications”和非空的“请先阅读｜安装与使用说明.txt”；说明必须覆盖当前系统要求、未公证包放行步骤、在线 AI 配置、平台登录、基础工作流、隐私边界和常见错误。
- 校验 DMG、App 签名、版本号、构建号、SHA-256 和实际启动进程。
- 使用 `otool -L` 和资源清单确认没有 Homebrew、`/usr/local`、源码目录或开发机绝对路径依赖。
- 从 DMG 安装到 `/Applications` 后验证，不以 `.build` 或 `dist/*.app` 直接运行代替。

## 2. 干净用户环境

优先使用另一台受支持 Mac、全新 macOS 用户或可还原虚拟机。测试开始前不得复制开发机的以下状态：

- `~/Library/Application Support/com.itou.chenggao`
- `~/Library/WebKit/com.itou.chenggao`
- UserDefaults、缓存、历史数据库、凭证文件和平台 Cookie
- shell 环境变量、Homebrew 运行库和源码目录

必须记录测试机器的 macOS 版本、CPU 架构、内存、是否首次安装以及是否存在旧版迁移数据。

## 3. 首次启动与基础功能

- 双击安装版可以启动，不崩溃、不要求开发工具或命令行依赖。
- 主窗口、设置页和所有工作区可打开；首次状态不引用开发机历史内容。
- 粘贴文本、保存历史和重新打开 App 后读取本机历史正常。
- 从一条有真实分镜和图片的历史记录点击“输出”，选择目录后应新建标题文件夹，并同时得到改写文稿、逐镜提示词文档和可用图片；缺图需明确标记，同名输出不得覆盖旧文件夹，文档不得泄露应用内部绝对路径或凭证。
- 错误提示必须说明真实原因，不能把未登录、缺 Key、网络失败或平台验证统一写成“页面结构变化”。

## 4. 在线 AI

- 安装包不包含任何主 Key 或测试 Key。
- 在干净机器通过设置页输入 endpoint、API Key，读取模型并选择模型。
- “保存并测试”成功后，用安装版完成一篇真实全文处理并检查结果页。
- 退出重开后配置仍可用；删除配置后不能继续误报在线可用。
- 若本次改动涉及图片生成：在设置页验证图片地址推导与独立地址保存；使用测试或有额度的兼容接口至少生成一张真实图片，确认 `image_url`／`url`／`b64_json` 中该服务实际返回的格式能保存、预览并重新打开。没有可用图片模型或额度时只能标记为接口候选，不能声称真实出图已通过。

## 5. 平台搜索

- 哔哩哔哩公共搜索在零登录、零历史状态下用真实关键词取得结果。
- YouTube 使用 Data API Key 或在 App 内重新登录后取得真实结果。
- X、TikTok、抖音、小红书和 Facebook 必须在该干净机器的 App 内重新登录；登录状态、搜索结果和退出重开后的会话复用都要验证。
- 视频号只验收分享链接导入，不冒充公开关键词搜索。
- 同时选择已就绪和未就绪平台时，已就绪平台继续执行，未就绪平台明确列出且不阻断整个任务。
- 第三方平台出现验证码、地区限制或账号权限不足时，应显示真实阻塞；这不允许通过打包开发者 Cookie 来绕过。

## 6. 视频与成稿

- 至少用一个真实公开视频链接验证字幕提取；无字幕时验证安装包内 Whisper 音轨转写。
- 验证英文、粤语或日语素材的原始识别保留源语言，最终修改稿输出简体中文。
- 从爆款研究结果进入拆解时，必须取得真实字幕或音轨，不能根据标题或简介生成正文。
- 修改稿、修改建议和配图提示词均来自当前配置的在线 AI；在线失败时显示错误，不伪造本地结果。

## 7. 对外发布判定

只有以上受影响项目在发行安装包和干净环境中通过，才可称为“可分发”。如果只在开发机完成，应标记为“本地候选包”。面向不受信任外部用户的公开版本还必须完成 Developer ID 签名、Hardened Runtime、Apple 公证、装订和 Gatekeeper 验证。

## 8. 界面与辅助功能

- 至少在安装版的“新建文稿、处理结果、爆款研究、平台账号、最近处理”五个工作区逐页检查：标题不重复、主要内容不溢出、列表与详情滚动互不干扰。
- 在接近最小窗口尺寸下检查搜索工具栏、AI 控制器、结果分页和右侧详情栏，不得依赖把窗口拉到超宽才能使用。
- 玻璃材质必须使用系统材料并保持文字对比度；不得用大面积不透明色块伪装玻璃。开启 macOS“减少透明度”时必须有可读的系统实色背景。
- 视觉更新不得擅自新增用户未要求的状态框、说明文字或悬浮按钮；装饰层必须关闭命中测试，不能阻挡输入、列表选择和滚动。
