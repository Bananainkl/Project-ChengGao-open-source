澄稿｜安装与使用说明
======================

一、使用条件

1. 当前安装包适用于 Apple 芯片 Mac（M1、M2、M3、M4 等），系统要求为 macOS 26.0 或更高版本。
2. 澄稿的正文改写与配图提示词依赖用户自己配置的在线 AI 服务；安装包不包含开发者或其他用户的 API Key。
3. 哔哩哔哩公共搜索通常无需登录。YouTube、X、TikTok、抖音、小红书、Facebook 等平台需要在本机澄稿中分别登录或配置相应接口；平台登录状态不会随安装包传给其他电脑。

二、安装澄稿

1. 双击“澄稿-版本号-macOS-arm64.dmg”。
2. 将“澄稿.app”拖到旁边的“Applications”文件夹。
3. 打开“访达”→“应用程序”，双击“澄稿”。

三、首次打开时提示无法验证开发者

当前内测包尚未使用 Apple Developer ID 签名和公证，因此 macOS 可能阻止第一次打开。这不代表需要关闭系统安全功能，也不要在终端执行来历不明的解除隔离命令。

方法一：

1. 在“应用程序”中按住 Control 键点按“澄稿”，选择“打开”。
2. 如果弹窗中出现“打开”按钮，再点一次“打开”。

如果仍被阻止，使用方法二：

1. 先尝试打开一次澄稿，然后关闭系统提示。
2. 打开“系统设置”→“隐私与安全性”。
3. 向下滚动到“安全性”区域，找到关于“澄稿”被阻止的提示。
4. 点击“仍要打开”或“允许打开”。
5. 使用 Mac 登录密码或 Touch ID 确认，再在弹窗中点击“打开”。

通常只需对这一版本执行一次。以后获得 Developer ID 和 Apple 公证后，将不再需要上述手动放行步骤。

四、首次配置在线 AI

1. 打开澄稿设置，进入“AI 服务”。
2. 选择 DeepSeek、智谱 GLM、MiniMax，或“自定义兼容接口”。
3. 自定义中转站可以填写站点根地址、以 /v1 结尾的地址，或者完整的 /v1/chat/completions 地址；应用会显示最终请求地址。
4. 输入自己的 API Key。
5. 点击“读取远程模型”，从服务返回的模型中选择一个；如果服务不提供模型目录，也可以手动输入模型 ID。
6. 点击“保存并测试连接”。只有出现连接成功后，才开始正式处理文章。

API Key 只保存在当前 Mac 用户的澄稿私有目录中，不进入文稿历史，也不会被写入安装包。请不要把完整 Key 发给他人或贴到公开聊天中。

五、新建文稿

1. 选择“粘贴文本”或“内容链接”。
2. 选择目标输出类型：
   - 短视频口播稿：强调开头钩子、自然口语和约 4 秒一个画面。
   - 公众号文章：生成导语、完整小节和 3–6 张 16:9 段落配图建议。
   - 小红书图文：生成适合卡片阅读的正文和 4–9 张 3:4 图片提示词；如果来源本身是图文，会结合原图文字和画面信息重新设计。
   - 视频号文案：信息链更完整、节奏更稳，约 6 秒一个画面。
3. 选择画面风格。风格会用于本次全部分镜、直接图片生成和导出的 ChatGPT Markdown，并随历史记录保存：
   - 自动匹配：根据现实、历史或抽象题材自动选择可信的视觉语言。
   - 电影纪实：真实材质、自然动作、环境光和克制电影调色。
   - 温暖日系手绘动画：柔和线稿、水彩背景、低饱和暖色和生活气息。
   - 经典欧美童话动画：圆润二维造型、丰富色彩、舞台式层次和童话光线。
   - 复古美式印刷漫画：粗黑墨线、半色调网点、有限复古色盘和戏剧性透视。
   - 夸张追逐喜剧动画：挤压拉伸形变、速度线、手绘背景和强动作节奏。
   - 毛线布偶：针织毛线、钩织线圈、羊毛毡、可见针脚和微缩布艺场景。
   - 粘土定格动画：粘土木偶、指纹和塑形痕迹、手工微缩布景和逐格姿态。
   - 手绘漫画：铅笔草线、墨线、排线或水彩、纸张纹理和手工上色。
   风格名称只描述视觉材料和语言，不代表或复刻任何具体工作室、影视作品、品牌或角色。
4. 选择输出语言。英语、粤语、日语等素材会先保留源语言完成识别，最终修改稿按所选语言输出；默认是简体中文。
5. 在输入框下方的“AI 控制器”选择模型。模型菜单会显示当前 AI 服务读取并缓存的远程模型；也可以从菜单刷新模型列表或打开完整 AI 设置。
6. 选择推理深度：
   - 自动：由模型和服务自行决定，不发送额外推理参数。
   - 快速：优先响应速度，适合简单整理。
   - 标准：兼顾速度与分析质量，适合一般改写。
   - 深入：加强事实覆盖、结构和遗漏检查，可能耗时更长并使用更多额度。
   如果兼容接口明确不支持推理深度参数，澄稿会自动移除该参数重试，并继续使用相应的编辑深度指令。
7. 点击“开始处理”。完成后前往左侧“处理结果”查看原稿、修改建议、修改稿和配图建议；配图页标题会显示本次采用的画面风格。

在线改写会核对原稿中的日期、数量、百分比、英文专名和引用名称。空格、全半角、引号及“日／号”等不改变事实的写法差异不会被误判为遗漏；若初稿确实漏掉事实，自动统稿会收到具体缺失项并逐项补回。

输入素材类型与输出类型相互独立。例如，小红书图文可以改成公众号文章或短视频口播稿；视频也可以改成信息更丰富的小红书图文。

六、生成或导出图片

（一）可选：通过中转站直接生成图片

1. 打开“设置”→“AI 服务”→“图片生成（可选）”。图片接口使用单独的图片 API Key，不会复用或覆盖上方的聊天 Key；即使两个接口使用同一枚 Key，也需要在图片区域单独填写并保存。
2. 如果中转站的图片接口与聊天接口位于同一地址，可以把“图片 API 地址”留空，澄稿会自动使用 /v1/images/generations；如果中转站另有图片地址，请填写它提供的完整 HTTP(S) 地址。地址栏不能填写 API Key。
3. 在“图片 API Key”输入框粘贴图片接口自己的 Key。Key 只写入权限为 0600、仅当前用户可读的独立凭证文件，不进入普通设置、历史或安装包。
4. 点击“读取图片模型”，从图片接口返回的远程模型中选择；如果中转站不提供模型目录，也可以手动填写图片模型 ID。读取模型只使用图片 Key，不使用聊天 Key。
5. 选择画面比例和图片质量，点击“保存图片生成设置”。
6. 完成文稿处理后，进入“处理结果”→“配图建议”，可以单独生成一张，也可以确认后批量生成全部图片。
7. 中转站必须真实支持图片生成；仅支持聊天、看图或返回文字提示词的模型不能直接出图。澄稿兼容 image_url、url 和 b64_json 三种常见返回方式。
8. 每张图片通常对应一次远程请求，可能产生费用。读取模型会发送一次带图片 Key 的目录请求；正式生图只在用户点击生成后调用，批量生成前会再次确认。
9. 如果曾误把 Key 填进图片地址栏，请立即在服务商后台撤销并换新。新版会拒绝保存疑似 Key，并清除旧版误存的地址字段，但无法代替服务商撤销已经暴露的凭证。

（二）可选：导出给 ChatGPT 批量生图

1. 进入“处理结果”或“最近处理”中的“配图建议”，点击“导出给 ChatGPT”，选择位置保存 .md 文档。此方式不要求在澄稿中配置图片生成接口。
2. 把导出的 Markdown 文档上传到支持文件读取与图片生成的 ChatGPT 会话。文档已包含全部图片的编号、比例、时间／位置、对应文案和完整提示词。
3. ChatGPT 应按编号逐项生成独立图片，不应把多个编号合并成拼图、九宫格或分镜表。不同提示词仍会对应多次图片生成，并非底层只发出一次图片请求。
4. 为适配 ChatGPT 可能单轮只完成前 10 张的实际行为，文档会主动按每 10 张一批组织任务。首次上传只执行第 1 批；完成后直接回复“继续”，ChatGPT 应从下一批开始，不需要重新上传文档或复制提示词。
5. 如果某一批在 10 张以内仍因额度、耗时或工具调用限制暂停，回复“继续”，ChatGPT 应先从本批首个未完成编号补齐，再进入下一批。

（三）输出完整短视频文件包

1. 文稿处理完成后，在“处理结果”或“最近处理”页面右上角点击“输出”，再选择一个用于存放文件包的目录。
2. 澄稿会在所选目录中自动新建一个以短视频标题命名的文件夹。标题中的斜杠、冒号等不能用于文件名的字符会自动替换；如果已有同名文件夹，会使用“标题 (2)”等新名称，不会覆盖原文件。
3. 文件包固定包含：
   短视频标题/
   ├── 改写文稿.md
   ├── 分镜与配图提示词.md
   └── 图片/
       ├── 01.png
       ├── 02.png
       └── ……
4. “改写文稿.md”保存修改后的完整文稿；“分镜与配图提示词.md”逐镜保存时间、对应文案、完整提示词和图片状态；“图片”子目录按镜头顺序收纳已经生成并仍可读取的图片。
5. 即使图片尚未全部生成，也可以输出文稿和全部分镜。缺少的图片不会被伪造，分镜文档会明确标记“未生成或原图片已不可用”，方便后续补图。
6. 图片批量生成仍在进行时，“输出”按钮会暂时停用；完成或停止生成后即可输出，避免把正在变化的半成品批次写入文件包。输出完成后，澄稿会自动在访达中定位新文件夹。

七、链接处理说明

1. 视频链接必须取得真实字幕或音轨。没有字幕时，澄稿使用安装包内的 Whisper 在本机转写，不会仅凭标题或简介编写正文。
2. 小红书视频会按视频处理；小红书图文会读取正文和可用原图。在线模型支持看图时会分析原图；纯文本模型则使用 macOS Vision 在本机提取图片文字和基础画面证据。
3. 如果平台要求登录、验证码或人机验证，请前往“平台账号”在澄稿的登录窗口中完成。澄稿不会代替用户绕过平台验证。
4. 小红书搜索页复制的 xsec_token 链接可能过期；内容也可能已删除或只对部分账号可见。遇到“链接可能已经过期”时，请在小红书中重新打开原笔记并复制最新分享链接。
5. 平台页面会不定期调整。如果已经登录但搜索或链接读取仍失败，请保留完整错误提示、关键词或原始链接，方便定位。

八、爆款研究

1. 输入关键词，选择已经具备搜索条件的平台，再选择时间范围和数量。
2. 哔哩哔哩支持公共搜索；其他平台按照页面提示完成登录或接口配置。
3. 视频号目前只支持粘贴分享链接，不提供公开关键词爆款搜索。
4. 选择搜索结果并进入拆解后，澄稿会保留你在新建文稿中选择的目标输出类型，不会因为来源是视频或图文而擅自切换格式。

九、常见问题

1. “在线模型没有返回完整、可用的改写稿”
   检查网络、API 余额、模型 ID和服务状态。先在“AI 服务”重新测试连接；较长或多图内容可能需要等待数分钟。

2. “需要登录”或搜索不到结果
   前往“平台账号”重新登录。换电脑、换 macOS 用户或移除账号后，都需要重新建立平台会话。

3. 应用打不开或立即退出
   确认电脑是 Apple 芯片、系统版本符合要求，并确认应用已拖入“应用程序”。如果 macOS 阻止打开，按第三部分在“隐私与安全性”中放行。仍然失败时，请记录 macOS 版本、Mac 芯片型号和弹窗原文。

4. 在线 AI 配置不会随安装包出现
   这是正常的安全设计。安装包不会内置、复制或分享任何人的 API Key；每台电脑必须由使用者自行配置。

5. “链接可能已经过期、内容已删除，或只对部分账号可见”
   这表示平台没有返回该条内容，不是 AI 改写失败。请确认当前账号确实能看到内容，并重新复制最新分享链接；已经删除或限制访问的内容无法由澄稿绕过平台权限读取。

十、隐私与安全

1. 正式在线改写会把标题、正文和必要的图片分析资料发送给用户所选择的在线 AI 服务。
2. 视频音轨的 Whisper 转写和兼容回退的 macOS Vision OCR 在本机完成。
3. 平台 Cookie 由 macOS WebKit 保存在当前用户环境中，不进入导出文件或安装包。
4. 不要把账号密码、验证码、Cookie 或完整 API Key 发给任何人。

版本更新后，请优先阅读安装包内的本说明和应用更新记录。
