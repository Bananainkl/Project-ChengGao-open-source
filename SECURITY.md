# Security Policy

## Reporting a vulnerability

Please do not disclose suspected vulnerabilities in a public issue.

Use GitHub's **Report a vulnerability** link on the repository Security page
when private vulnerability reporting is available. Include the affected
version, reproduction steps, impact, and any suggested mitigation.

If private reporting is unavailable, use the contact options on the
[Bananainkl GitHub profile](https://github.com/Bananainkl) and share only enough
non-sensitive detail to arrange a private report.

The maintainer will acknowledge a complete report when practical, investigate
it, and coordinate a fix and disclosure. Please allow reasonable time for a
patch before publishing technical details.

## Supported versions

Security fixes are made on the latest released version and the default branch.
Older versions may not receive backports.
