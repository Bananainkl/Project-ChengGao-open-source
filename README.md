# 澄稿

当前版本：`1.19.1 (52)`。公开版本由根目录 `VERSION` 管理，内部构建号由
`BUILD_NUMBER` 管理；每次功能更新必须先递增二者，再重新生成安装包。

一个面向 macOS 26 的内容提取与改写应用。目标是在 Apple Silicon Mac 上完成网页读取、
字幕／正文整理和在线全文改写。用户配置自己的 OpenAI 兼容接口；视频无字幕时仍由包内 Whisper
在本机转写。可选图片接口复用同一提供商的 Key，并兼容中转站返回 `image_url`、`url` 或 `b64_json`。

## 当前状态

- 原生 SwiftUI `NavigationSplitView`、检查器、设置场景与菜单命令
- macOS 26 Liquid Glass 编辑器、结果卡片和主操作按钮
- 普通网页正文的本地提取；公众号链接优先锁定正文容器
- 爆款研究中心：关键词搜索、时间范围、热度排行、SQLite 去重与 CSV／JSON／Markdown 导出
- 哔哩哔哩使用快速公共搜索；YouTube 可用 Data API 或网页登录搜索，其他平台使用登录后的真实网页尝试搜索
- 登录窗口与搜索复用同一个长期 WebView，保留 JavaScript 会话与浏览器状态；不保存明文账号、密码、Cookie 或 Token
- 研究结果可一键进入真实字幕／音轨提取、实质改写和镜头提示词流程
- 自动过滤二维码、关注引导、点赞在看、阅读原文、服务宣传和页面推荐等噪声
- 网页文章按核心事实、论点、数据与案例重新组织，不机械复述页面上的全部文字
- B 站完整字幕提取；无公开字幕时自动下载音轨并在本机转写
- 内嵌 whisper.cpp v1.9.1 与 Whisper Small Q5_1 多语言语音模型
- 完全离线的规则整理兜底
- App 内嵌 `llama.cpp b10015` 运行时与 Qwen3 1.7B Q4_K_M GGUF
- 双模型自适应：Qwen3 1.7B Q4 极速档 + Qwen3 4B Q4_K_M 增强档
- 自动模式在 8GB 设备保留 1.7B，在 16GB 及以上设备启用 4B
- 可选 OpenRouter 在线优先模式；密钥保存在 macOS 钥匙串，断网、限流或在线质量不合格时自动回退到本地模型
- 语音识别后先执行上下文校对，记录同音词和专有名词的“原词 → 校正词 → 原因”
- 可选中文维基百科词条核验，仅发送候选专有名词，不上传完整原稿
- 四页独立结果视图：“完整原稿 → 逐段并排修改建议 → 修改后全文 → 分段配图建议”
- 默认使用简体中文，可在处理前切换为繁体中文；生成后再执行一次简繁规范化
- 修改稿页默认打开，处理完成后自动定位结果；四个结果分页使用固定高度的独立滚动视口
- 支持按页复制、复制反馈、逐段进度和停止本地模型生成
- 改写采用“全文编辑蓝图 → 带前后文的分段改写 → 全文统稿与独立审稿”，短视频约 300 字一个语义块
- 质量门逐段检查事实覆盖、日期／数字／信源锚点、全文三区覆盖、结构和相似度；遗漏后半篇时不会交付
- 最近处理可回看完整四页结果；最近 50 篇保存到 Application Support
- 短视频按每 3–5 秒一个镜头给出时间码、对应口播和可直接复制的图像 AI 提示词
- 长口播分段调用本地模型，原稿始终逐字保留，不受 4K 上下文截断
- 无字幕、无可用音轨或转写过短时硬失败，不根据标题或简介编造正文
- 小模型结构化输出失败时自动生成基础整理稿，确保修改稿页不会只显示修改意见
- 固定 SHA-256 的运行时、模型和许可证引导脚本
- 2K/4K/8K 上下文可选；8GB 设备自动限制为最高 4K
- SwiftPM `.app` 打包、Codex Run action 和自动验证脚本

当前应用直接启动包内的 `llama-cli` 子进程并读取独立输出文件，不启动服务器、不开放端口，
也不要求用户安装 Ollama。`EmbeddedModelRuntime` 把视图层与运行时隔开；如果模型资源不存在，
应用会自动退回离线规则整理。

视频链接先读取平台字幕。若字幕为空，应用下载视频音轨，使用 macOS 自带 `afconvert` 转成
16kHz WAV，再通过包内 whisper.cpp + Small Q5_1 在进程内转写。转写模型释放后才启动 Qwen，
所以两个模型不会同时驻留。

## 模型决策

设置中的“在线优先 · 自动离线回退”使用 OpenRouter 的免费模型路由处理全文。免费路由的实际模型
会随当时可用性变化，结果备注会记录本次实际路由；使用在线模式会把标题和全文发送到第三方服务。
不希望上传内容、没有网络或尚未配置密钥时，可继续选择自动、极速或增强本地模式。

## 爆款研究

侧边栏进入“爆款研究”，输入关键词并选择时间范围后即可取得当前候选内容。哔哩哔哩公共搜索
无需登录；YouTube 可直接复用网页登录会话，也可在“平台账号”保存 Data API Key 以优先使用官方接口。
搜索结果按照需求文档中的对数互动指标与 30 天新鲜度衰减计算综合热度；不同平台的数据口径不同，
跨平台分数只作为选题参考。

选择视频后点击“拆解并生成爆款文稿”，澄稿会先取得真实字幕；没有可用字幕时，仅在平台允许且
能取得音轨的情况下使用本地 Whisper 转写。无法取得完整内容时会停止，不会根据标题或简介生成正文。
X、TikTok、抖音、小红书、Facebook 和视频号会复用账号页的长期 WebView，在真实搜索页提取渲染结果。
这条路径依赖有效登录和平台当前页面结构；验证码、反自动化校验或 DOM 改版会明确报错。账号页只把
真实认证 Cookie 视为已登录，抖音的匿名 CSRF Cookie 不再造成假登录状态。

8GB 设备的默认模型为 Qwen3 1.7B Q4；16GB 及以上设备在自动模式使用 Qwen3 4B Q4_K_M：

- 明确覆盖简体中文、繁体中文与粤语
- Apache 2.0，应用分发条件清晰
- 4K 上下文下预计峰值约 2.7GB，保留足够的 8GB 系统余量

Gemma 3 1B Q4 保留为可选适配目标。它更小，但 1B 版本更偏英文；Gemma 3 的完整
多语言定位从 4B 起更有代表性，而 4B 不适合作为 8GB 设备的默认模型。

增强模式使用官方 Qwen3 4B Q4_K_M（约 2.5GB）。它与 1.7B 共用同一个 llama.cpp
运行时和 Apache 2.0 许可。用户可选择自动、极速或增强；即使手动选择增强，8GB 设备
也会回落到 1.7B 保护模式，避免为了模型档位牺牲系统流畅度。

## 构建和运行

首次准备运行时、模型和许可证：

```bash
./script/bootstrap_runtime.sh
```

```bash
./script/build_and_run.sh
```

验证应用构建、打包和进程启动：

```bash
./script/build_and_run.sh --verify
```

运行测试：

```bash
swift test
```

产物位于 `dist/澄稿.app`。

生成带“拖入 Applications”入口的本地 DMG 安装包：

```bash
./script/package_dmg.sh
```

安装包名称会自动带版本号，例如 `dist/澄稿-1.9.1-macOS-arm64.dmg`，并同时生成
SHA-256 校验文件。DMG 根目录会自动附带“请先阅读｜安装与使用说明.txt”，包含未公证
内测包的 macOS 放行步骤、AI 配置、平台登录、基础工作流和隐私说明；缺少该文件时打包脚本会停止。
面向其他用户公开分发时，使用 `script/package_release.sh` 完成
Developer ID 签名与 Apple 公证；本机没有相应证书时只能生成本地签名安装包。

每次更新的完整本地发布流程统一运行：

```bash
./script/release_local.sh
```

该命令会要求 `CHANGELOG.md` 已包含当前版本，随后重新测试构建并生成带版本号的 DMG，
再以可回滚方式替换 `/Applications/澄稿.app`，验证新版本能够启动后才删除旧版本备份。

本机实测：

- 1.28GB Q4_K_M 模型
- 4K 上下文
- 短中文素材端到端生成约 2–8 秒，取决于输出长度
- 直接运行时峰值 RSS 约 1.86GB
- App 空载 RSS 约 128MB
- Whisper Small Q5_1 权重约 190MB；343 秒中文视频本机转写实测约 15 秒（M3 Pro）

长文本仍按 2.7GB 峰值预算设计，不能用短样本测量替代 M1 8GB 真机验收。

## 8GB 运行约束

- 默认上下文 4096 tokens；16GB 设备可选 8192 tokens
- 单任务串行推理
- 模型权重使用 Q4 量化和内存映射
- 语音转写模型与文稿模型不同时驻留
- 每个 llama.cpp 子进程完成后自动释放模型；用户可随时停止当前生成
- 长内容按原始段落分块提出建议并修改，最后无损拼接
